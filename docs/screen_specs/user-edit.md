# ユーザー編集画面 仕様

対応モックアップ: [docs/mockups/user-edit.html](../mockups/user-edit.html)

## 1. 画面概要

ユーザー（`AspNetUsers`）の新規登録、またはユーザー名・パスワードの変更を行う画面。
[ユーザー管理画面](users.md)から「新規登録」または「パスワード変更」で遷移する。

---

## 2. 入力項目

| 項目 | 種別 | 内容 |
|------|------|------|
| ユーザー名 | テキスト入力 | `IdentityUser.UserName`。必須。既存ユーザー名（自分自身を除く）と重複不可 |
| パスワード | パスワード入力 | 新規登録時は必須。編集時は空欄であればパスワードを変更しない |
| パスワード（確認） | パスワード入力 | パスワードを入力した場合、一致を必須とする |

新規登録時はすべて空欄、編集（パスワード変更）時はユーザー名のみプレフィルし、パスワード欄は空欄とする。

---

## 3. ボタン動作

| ボタン | 動作 |
|--------|------|
| 保存する | 4章の入力チェックを行い、OKなら5章の保存処理を実行。完了後トーストで通知し、[ユーザー管理画面](users.md)へ遷移する |
| キャンセル | 確認なしで[ユーザー管理画面](users.md)へ遷移する（未保存の変更は破棄） |

---

## 4. 入力チェック

- ユーザー名: 必須（未入力はエラー）。自分自身を除く既存ユーザー名と重複しないこと
- パスワード: 新規登録時は必須。Identityのデフォルトパスワード要件（最低6文字、英数字混在等）に従う
- パスワード（確認）: パスワードを入力した場合、パスワードと一致しないとエラー

---

## 5. 保存処理

- 新規登録時: `UserManager.CreateAsync`でユーザーを新規作成し、`CreatedAt`に現在時刻を設定する。
- 編集時:
  - ユーザー名が変更されている場合、`UserManager.SetUserNameAsync`で更新する。
  - パスワードが入力されている場合のみ、`UserManager.RemovePasswordAsync` → `AddPasswordAsync`（またはトークンベースの`ResetPasswordAsync`）でパスワードを更新する。

---

## 6. 確定事項

- パスワードの強度要件はASP.NET Core Identityのデフォルト設定のまま使用する（緩和しない）。4章の記載のとおり
