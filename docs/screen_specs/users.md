# ユーザー管理画面 仕様

対応モックアップ: [docs/mockups/users.html](../mockups/users.html)

## 1. 画面概要

システムにログイン可能なユーザー（`AspNetUsers`、ASP.NET Core Identity）の一覧表示・新規登録・削除を行う画面。
このシステムは基本的に1人で利用するが、ユーザー登録機能自体は提供する。全ユーザーが全機能を利用可能（ロールによる権限分けは行わない）。
パスワード変更・ユーザー名変更は[ユーザー編集画面](user-edit.md)で行う。

---

## 2. 一覧

### 2.1 列定義

| 列 | データソース / 算出方法 |
|----|------|
| ユーザー名 | `IdentityUser.UserName`。ログイン中のユーザー自身の行には「（ログイン中）」を付記する |
| 登録日 | ユーザー作成日時（`ApplicationUser.CreatedAt`。後述） |
| 操作 | パスワード変更・削除リンク（2.3参照） |

並び順: 登録日昇順（登録順）。

> Identityの`IdentityUser`には作成日時の標準カラムが無いため、`ApplicationUser : IdentityUser`を作成し`CreatedAt`プロパティを追加する。

### 2.2 ボタン動作（一覧上部）

| ボタン | 動作 |
|--------|------|
| + 新規登録 | [ユーザー編集画面](user-edit.md) を新規登録モードで開く |

### 2.3 行ごとの操作

| 操作 | 動作 |
|------|------|
| パスワード変更 | [ユーザー編集画面](user-edit.md) に該当ユーザーを指定して遷移し、ユーザー名をプレフィルする（パスワード欄は空欄） |
| 削除 | 確認ダイアログを表示し、OKなら3章の判定に従い削除する |

---

## 3. 削除処理

以下のいずれかに該当する場合、「削除」リンクを無効化（クリック不可）し、削除を行わせない。

| 条件 | 理由 |
|------|------|
| 削除対象がログイン中の自分自身である | 自分自身を削除するとログイン不能になる |
| 登録されているユーザーが1人のみである | システムにログイン可能なユーザーが0人になることを防ぐ |

上記に該当しない場合のみ削除可能。削除してもそのユーザーが過去に作成・更新したデータ（`CreatedBy`/`UpdatedBy`に記録されたユーザー名文字列）はそのまま残る。
