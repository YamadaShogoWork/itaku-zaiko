# 色マスタ管理画面 仕様

対応モックアップ: [docs/mockups/color.html](../mockups/color.html)

## 1. 画面概要

色（`Color`）マスタの登録・編集・削除を1画面で行う。一覧の各行をインライン編集する形式とし、登録・編集用の別画面は設けない。

`Size`マスタ（`Size`テーブル・`Product.SizeId`）はDB設計上保持するが、現状未使用のため本画面（および他画面）にサイズの管理UIは設けない（[product.md](product.md)・[product-edit.md](product-edit.md)と同様の方針）。

---

## 2. 一覧

### 2.1 列定義

| 列 | データソース / 算出方法 |
|----|------|
| 色名 | `Color.ColorName` |
| 使用商品数 | 該当`ColorId`を参照する`Product`の件数 |
| 操作 | 編集・削除リンク（2.3参照） |

並び順: `Color.ColorId` 昇順（登録順）。

### 2.2 新規追加

- 一覧上部の「+ 新規追加」ボタンを押すと、一覧の先頭に色名が空欄の編集行を追加する（フォーカスを当てる）。
- 編集行の「操作」列は「保存」「キャンセル」リンクを表示する。
- 「保存」: 3章の入力チェックを行い、OKなら`Color`を新規作成して一覧を再読込する。
- 「キャンセル」: 追加行を破棄し、通常表示に戻す。

### 2.3 行ごとの操作

| 操作 | 動作 |
|------|------|
| 編集 | その行の色名を入力欄に切り替え、現在値をプレフィルする。「操作」列を「保存」「キャンセル」リンクに切り替える |
| 保存（編集行） | 3章の入力チェックを行い、OKなら`Color.ColorName`を更新し、通常表示に戻す |
| キャンセル（編集行） | 変更を破棄し、通常表示に戻す |
| 削除 | 確認ダイアログを表示し、OKなら4章の判定に従い削除する |

---

## 3. 入力チェック

- 色名: 必須（未入力はエラー）
- 色名: 他の`Color`レコードと重複しないこと（一意制約。重複時はエラー）

---

## 4. 削除処理

`Client`の`IsActive`・`Product`の削除制限と同様、`Color`にも無効化フラグは設けない。「削除」操作時、対象`Color`について以下を判定する。

| 判定 | 条件 | 動作 |
|------|------|------|
| 削除可 | 該当`ColorId`を参照する`Product`が存在しない | `Color`レコードを削除する |
| 削除不可 | 該当`ColorId`を参照する`Product`が1件以上存在する | 削除せず、エラーメッセージ「この色は商品で使用されているため削除できません」を表示する |

---

## 5. 確定事項

- 削除不可時のエラーメッセージ: 「この色は商品で使用されているため削除できません」（[product.md](product.md) ❓2と統一）
