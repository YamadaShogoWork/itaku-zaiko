# 売上報告履歴画面 仕様

対応モックアップ: [docs/mockups/sales-report-history.html](../mockups/sales-report-history.html)

## 1. 画面概要

これまでに登録された売上報告（`SalesReport`）を一覧で確認し、編集・削除・委託販売納品書の再出力を行う画面。
編集は [売上報告入力画面](sales-report.md) に取引先・対象年月を指定して遷移し、既存の期末在庫数のプレフィル（[sales-report.md 4.2](sales-report.md)）を利用する。
再出力は、各行の「帳票」リンクからモーダルで委託販売納品書（Excel、全項目）のプレビューを表示する（4章参照）。[売上報告入力画面](sales-report.md) の保存時に同内容の帳票が一度出力されているため、モーダルからの出力は**再出力専用**である。月次繰越処理（[CLAUDE.md](../../CLAUDE.md) 月次繰越処理）は実行しない。

## 3. 一覧

### 3.1 表示単位・列定義

1行 = 1登録単位（`ClientId` × `YearMonth` の組み合わせ）。

| 列 | データソース / 算出方法 |
|----|------|
| 対象年月 | `YearMonth` を「YYYY年M月」形式で表示 |
| 取引先 | `Client.ClientName` |
| 品目数 | 同一グループに属する `SalesReport` レコード数 |
| 売上点数 | 同一グループの各 `SalesReport` レコードについて算出した売上点数（[sales-report.md 4.2](sales-report.md)：期首在庫+納品計−期末在庫）の合計 |
| 売上額 | 同一グループの各レコードの売上額（売上点数×下代）の合計 |
| 操作 | 編集・削除リンク（3.2参照） |

並び順: `YearMonth` 降順、同年月内は `ClientId` 昇順。

### 3.2 ボタン動作

| 操作 | 動作 |
|------|------|
| 編集 | [売上報告入力画面](sales-report.md) に該当行の取引先・対象年月をクエリパラメータ等で指定して遷移し、4.2のプレフィル付きで表示する |
| 帳票 | モーダルで委託販売納品書（Excel、全項目）のプレビューを表示する（4章参照） |
| 削除 | 確認ダイアログを表示し、OKなら該当グループ（`ClientId` × `YearMonth`一致）の `SalesReport` レコードをすべて削除し、一覧を再読込する |

### 3.3 翌月分が登録済みの場合（編集・削除不可）

同一取引先で翌月（`YearMonth` の翌月）の `SalesReport` が1件以上存在する場合、その行の「編集」「削除」リンクは無効化（`action-link disabled`）し、隣に「翌月分登録済み」のラベルを表示する。

- 編集を無効化する理由: 遷移先の[売上報告入力画面](sales-report.md)は8章の読み取り専用表示になり、実質的に編集できないため
- 削除を無効化する理由: 当月の `SalesReport.ClosingStock` は月次繰越処理により翌月の `Delivery`（`IsCarryOver=true`）として登録済みであり、翌月の `SalesReport` の期首在庫の根拠データとなっている。削除すると翌月のデータと整合しなくなるため

修正する場合は、翌月以降の売上報告を新しい月から順に削除していく必要がある。

「帳票」リンクは翌月分の登録有無に関わらず常に有効である（再出力に影響しないため）。

## 4. 帳票プレビューモーダル

### 4.1 概要

一覧の「帳票」リンク押下で表示するモーダル。選択した行（`ClientId` × `YearMonth`）の `SalesReport` データから委託販売納品書（Excel、全項目）のプレビューを表示する。[売上報告入力画面](sales-report.md) の保存時に同内容の帳票が一度出力されているため、本モーダルは**再出力専用**であり、月次繰越処理は実行しない。

### 4.2 ヘッダー情報

| 項目 | データソース |
|------|------|
| 取引先名 | `Client.ClientName` |
| 対象期間 | 対象年月の1日〜末日を「YYYY年M月D日 〜 YYYY年M月D日」形式で表示 |

### 4.3 表示対象行・列定義

選択中の取引先・対象年月に `SalesReport` レコードが存在する商品のみを表示する。
[売上報告入力画面 4.1](sales-report.md) と同じグルーピング（`Product.ProductName` でグルーピングし、`ColorId` 昇順でrowspan）を適用する。

[requirements.md 3-3 帳票の列構成](../requirements.md) に準拠する。

| 列 | データソース / 算出方法 |
|----|------|
| 品名 | `Product.ProductName` |
| 色 | `Color.ColorName`。`Product.ColorId` が null の場合は「—」表示 |
| 上代 | `Product.RetailPrice` |
| 下代 | `Product.RetailPrice` × `ClientProduct.CommissionRate` |
| 期首在庫 | `Delivery`（`IsCarryOver=true`）の `Quantity` |
| 期間内納品数（最大4列） | `Delivery`（`IsCarryOver=false`）を納品日ごとに展開。列見出しは納品日（M/D形式） |
| 納品計 | 期間内納品数の合計 |
| 期末在庫 | `SalesReport.ClosingStock` |
| 売上点数 | 期首在庫 + 納品計 − 期末在庫 |
| 売上額 | 売上点数 × 下代 |

### 4.4 合計行

品名〜期首在庫を結合して「合計」と表示し、納品計・期末在庫・売上点数・売上額は各列の合計を表示する（期間内納品数の列は空欄）。

### 4.5 ボタン動作

| ボタン | 動作 |
|--------|------|
| Excel出力 | 表示中のプレビューと同内容の委託販売納品書（Excel）を生成し、ブラウザでダウンロードする。月次繰越処理は実行しない |
| 閉じる | モーダルを閉じる |

---

## 5. 確定事項

- 検索条件「取引先」のデフォルト選択値: 未選択（空欄）。取引先を選ぶまでデータを表示しない。
- Excel出力ファイル名規則: `委託販売納品書_{取引先名}_{YYYYMM}.xlsx`（[sales-report.md](sales-report.md) 9章と統一）。
- サイドバーのナビゲーション配置: 「業務」セクション内、「売上報告入力」の直後に「売上報告履歴」を配置する。
- 削除時の月次繰越への影響（旧❓1）: 翌月の `SalesReport` が存在する場合は3.3の通り削除自体をブロックするため、繰越 `Delivery` との連動削除は発生しない。
