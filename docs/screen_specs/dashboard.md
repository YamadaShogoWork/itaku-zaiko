# ダッシュボード画面 仕様

対応モックアップ: [docs/mockups/dashboard.html](../mockups/dashboard.html)

## 1. 画面概要

ログイン後の最初に表示するトップ画面。当月のサマリー・在庫アラート・取引先別売上ランキング・最近の納品履歴を表示する。

対象年月は常にシステムの現在日時の年月（例：2026-05）とし、選択UIは設けない。

---

## 2. ヘッダー

| 項目 | 内容 |
|------|------|
| タイトル・サブタイトル | 「ダッシュボード」「{対象年月} 現在の状況」 |
| 売上報告履歴ボタン | [売上報告履歴画面](sales-report-history.md)へ遷移する |

---

## 3. サマリーカード

| カード | データソース / 算出方法 |
|--------|------|
| 取引先数 | `Client.IsActive=true` の件数 |
| 今月の納品件数 | 対象年月内に`DeliveredAt`がある`Delivery`（`IsCarryOver=false`）を「取引先×納品日」単位でグルーピングした件数（[delivery-history.md](delivery-history.md)の一覧と同じ単位） |
| 今月の売上合計 | 対象年月の`SalesReport`（`YearMonth`=対象年月）について、(期首在庫+期間内納品計−期末在庫)×下代 を全`SalesReport`分合計した値。対象年月のSalesReportが1件も無い場合は¥0と表示する |
| 在庫アラート | 6章の在庫アラートロジックで「危険」または「警告」と判定された(取引先×商品)の件数 |

- 「今月の売上合計」の下部に前月比（%）を表示する。前月比 = (今月の売上合計 − 前月の売上合計) ÷ 前月の売上合計 × 100。前月の売上合計が0の場合は前月比を表示しない（「—」表示）。
- 「在庫アラート」カードの数値は危険件数が1件以上あれば赤字、無ければ通常色で表示する。

---

## 4. クイックアクション

| ボタン | 動作 |
|--------|------|
| 納品を登録する | [納品登録画面](delivery.md)へ遷移 |

## 5. 取引先別売上ランキング

- 3章「今月の売上合計」と同じ対象年月のデータを使用し、取引先（`Client`）ごとに売上額を合計して降順に表示する。
- 上位4件を表示し、5件目以降は合計して「その他」として最終行に表示する（取引先が4件以下の場合は「その他」行を表示しない）。
- 各行のバー（`.rank-bar`）の幅は、1位の売上額を100%とした相対値で表示する。

---

## 6. 在庫アラート

### 6.1 現在庫の算出（[requirements.md](../requirements.md) 3-4と同じ）

- 対象（取引先×商品）について、直近の`SalesReport`が存在する場合：
  `現在庫 = その SalesReport.ClosingStock + （対象月より後に登録された IsCarryOver=false の Delivery の Quantity 合計）`
- 直近の`SalesReport`が存在しない場合：
  `現在庫 = それまでに登録された Delivery（IsCarryOver問わず）の Quantity 合計`

### 6.2 アラート区分

| 現在庫 | 区分 | 表示 |
|--------|------|------|
| 1〜2 | 危険 | 赤丸（`.dot-danger`）、在庫数を赤字表示 |
| 3〜5 | 警告 | 橙丸（`.dot-warning`）、在庫数を通常のアクセントカラーで表示 |
| 6以上 | アラート無し | 一覧に表示しない |

### 6.3 表示内容

| 列 | データソース |
|----|------|
| 商品 | `Product.ProductName` + 色（[商品名の表示仕様](../requirements.md)に従って連結） |
| 取引先 | `Client.ClientName` |
| 在庫数 | 6.1の現在庫 |

- `IsActive=false`の取引先は対象外とする。
- 現在庫が少ない順（昇順）に表示し、上位5件まで表示する。
- カード見出しのバッジ「残り5点以下」は固定文言。
- カード下部に凡例（「● 赤：残り1〜2点　● 橙：残り3〜5点」）を表示する。

---

## 7. 最近の納品履歴

1行 = 1登録単位（`ClientId` × `DeliveredAt` × `IsCarryOver` の組み合わせ）。[納品履歴画面 3.1](delivery-history.md)と同じ表示単位とする。

| 列 | データソース / 算出方法 |
|----|------|
| 納品日 | `Delivery.DeliveredAt` |
| 取引先 | `Client.ClientName` |
| 品目数 | 同一グループに属する `Delivery` レコード数 |
| 合計数量 | 同一グループの `Delivery.Quantity` 合計 |
| 種別 | `IsCarryOver=true` → 「繰越」（`badge-warning`）、`false` → 「通常」（`badge-info`） |

- `DeliveredAt`降順、同日内は`CreatedAt`降順でグルーピングし、直近5件（グループ単位）を表示する。
- 操作列（編集・削除）は設けない。編集・削除は[納品履歴画面](delivery-history.md)で行う。
- 「すべて見る」リンクは[納品履歴画面](delivery-history.md)へ遷移する。
