# 納品登録画面 仕様

対応モックアップ: [docs/mockups/delivery.html](../mockups/delivery.html)

## 1. 画面概要

取引先・納品日を選択し、その取引先の取扱商品の納品数量をグリッド形式で一括登録する画面。
登録すると同時に委託販売納品書（Excel）を出力する。

過去の登録内容の確認・編集・削除は [納品履歴画面](delivery-history.md) から行う（編集時はこの画面に取引先・納品日を指定して遷移し、4.2のプレフィルが適用される）。

---

## 2. 検索条件エリア（入力テーブル用）

| 項目 | 種別 | データソース / 内容 |
|------|------|------|
| 取引先 | セレクトボックス | `Client` 全件を `ClientName` で表示（値は `ClientId`） |
| 納品日 | 日付入力（`type=date`） | デフォルト値は本日 |
| 表示する（ボタン） | ボタン | 選択した取引先・納品日で入力テーブルを再読込（GET） |

---

## 3. ステータスバー

| 項目 | データソース / 算出方法 |
|------|------|
| 取引先 | 選択中の `Client.ClientName`（検索条件のエコー表示） |
| 納品日 | 選択中の納品日を「YYYY年M月D日」形式で表示 |
| 入力中 N / M 件 | M = 入力テーブルの表示行数、N = 納品数量が1以上の行数（バッジ表示） |

---

## 4. 入力テーブル

### 4.1 表示対象行

- 売上報告入力画面と同じロジックを適用する。
  - 選択中の取引先（`Client`）の `ClientProduct` に登録されている `Product` のみを表示する。
  - `Product` を `Color`（`Product.ColorId` で JOIN、null可）と結合する。
  - `Product.ProductName` でグルーピングし、同一品名内は `ColorId` 昇順でソート（rowspanでまとめる）。

### 4.2 既存データのプレフィル

- 選択した取引先 × 納品日について、`Delivery`（`IsCarryOver=false`）に既に登録済みのレコードが存在する場合、該当商品の「納品数量」欄にその `Quantity` をプレフィルする（編集モード）。
- 該当レコードが存在しない商品の「納品数量」欄は空欄（プレースホルダ「0」）。

### 4.3 列定義

| 列 | データソース | 算出方法 |
|----|------|----------|
| 品名 | `Product.ProductName` | 色・サイズは連結しない（売上報告入力画面と同じ例外仕様） |
| 色 | `Color.ColorName` | `Product.ColorId` が null の場合は「—」表示 |
| 現在庫 | 計算値 | ダッシュボードの在庫アラートと同じロジック（[3-4](../requirements.md)参照）。`現在（今）`時点で算出し、選択中の納品日には依存しない。<br>直近の`SalesReport`が存在する場合：`SalesReport.ClosingStock` + （対象月より後に登録された `IsCarryOver=false` の `Delivery.Quantity` 合計）<br>存在しない場合：それまでに登録された `Delivery`（`IsCarryOver`問わず）の `Quantity` 合計 |
| 納品数量 | 入力欄（`number`, `min=0`） | 4.2のプレフィル、または手入力 |

### 4.4 合計行

| 列 | 内容 |
|----|------|
| 品名〜現在庫 | 結合して「合計」と表示 |
| 納品数量 | 各行の「納品数量」の合計（空欄は0として扱う） |

### 4.5 リアルタイム計算

納品数量欄の入力（input/blur）のタイミングで、JavaScriptにより以下を再計算・更新する。

- 合計行の「納品数量」
- ステータスバーの「入力中 N / M 件」

---

## 5. ボタン動作

| ボタン | 動作 |
|--------|------|
| 表示する | 選択中の取引先・納品日で入力テーブルを読み込み直す（4.2のプレフィルを適用） |
| クリア | 確認ダイアログを表示し、OKなら入力中の納品数量欄をすべて空にする（画面表示のみ、未保存） |
| 登録してExcelを出力する | 確認ダイアログを表示し、OKなら6章の入力チェックを行い、7章の保存処理を実行。完了後トーストで完了を通知し、8章のExcelをダウンロードする。入力テーブルを再読込し、同画面に留まる |

---

## 6. 入力チェック

- 納品数量は0以上の整数（空欄は0として扱う）。
- 選択した取引先の対象年月（納品日の年月）について、既存の `Delivery`（`IsCarryOver=false`）の納品日の種類が既に4日分存在し、かつ今回の納品日がそのいずれにも含まれない場合はエラーとする（「この月の納品は既に4回登録されています。帳票の列数が上限に達しているため、新たな納品日では登録できません」）。同一納品日への再登録（上書き）はエラーにならない。

---

## 7. 保存処理

表示中の全商品行について、以下のルールで `Delivery`（`ClientId`・`ProductId`・`DeliveredAt`=選択した納品日・`IsCarryOver=false`）をUPSERT/削除する。

| 入力後の数量 | 既存レコード | 動作 |
|---|---|---|
| 1以上 | あり | `Quantity` を更新 |
| 1以上 | なし | 新規作成 |
| 0 | あり | 削除（納品の取消として扱う） |
| 0 | なし | 何もしない |

同一の取引先×納品日に対して再度この画面で登録する場合は、常に4.2のプレフィル＋本処理による「編集」として扱う（新規登録と編集を画面上で区別しない）。

---

## 8. Excel出力（委託販売納品書・納品時）

要件定義 [3-3 #1](../requirements.md) の仕様に従う。

- 期間内納品数列のうち、選択した納品日の列に保存後の納品数量を記載する。
- 期首在庫数（繰越分）は参考として記載する。
- 期末在庫数・売上点数・売上額は空欄とする。
- 月次繰越処理は実行しない。
- 帳票の詳細な列構成・テンプレートは別途定義する（本書のスコープ外）。

---

## 9. 確定事項

- 検索条件「取引先」のデフォルト選択値: 未選択（空欄）。取引先を選ぶまでデータを表示しない。
