# 取引先一覧画面 仕様

対応モックアップ: [docs/mockups/client.html](../mockups/client.html)

## 1. 画面概要

登録済みの取引先（`Client`）を一覧表示し、新規登録・編集・削除（無効化）を行う画面。
登録・編集は [取引先編集画面](client-edit.md) で行う。

---

## 2. 一覧

### 2.1 列定義

| 列 | データソース / 算出方法 |
|----|------|
| 取引先名 | `Client.ClientName` |
| FAX番号 | `Client.FaxNumber` |
| 取扱商品数 | 該当 `Client` に紐づく `ClientProduct` の件数 |
| 状態 | `IsActive=true` → 「有効」バッジ（`badge-success`）、`false` → 「無効」バッジ（`badge`） |
| 操作 | 編集・削除/有効化リンク（2.3参照） |

並び順: `ClientId` 昇順（登録順）。`IsActive=false` の取引先も一覧から除外せず表示する。

### 2.2 ボタン動作（一覧上部）

| ボタン | 動作 |
|--------|------|
| + 新規登録 | [取引先編集画面](client-edit.md) を新規登録モードで開く |

### 2.3 行ごとの操作

| 状態 | 操作 | 動作 |
|------|------|------|
| 有効 | 編集 | [取引先編集画面](client-edit.md) に該当 `ClientId` を指定して遷移し、既存データをプレフィルする |
| 有効 | 削除 | 確認ダイアログを表示し、OKなら3章の判定に従い削除または無効化し、一覧を再読込する |
| 無効 | 編集 | 有効な取引先と同様に編集画面へ遷移する |
| 無効 | 有効化 | 確認ダイアログ無しで `IsActive=true` に更新し、一覧を再読込する |

---

## 3. 削除処理

「削除」操作時、対象 `Client` について以下を判定する。

| 判定 | 条件 | 動作 |
|------|------|------|
| 物理削除 | 該当 `ClientId` を参照する `Delivery`・`SalesReport`・`ClientProduct` のいずれも存在しない | `Client` レコードを削除する |
| 無効化 | 上記のいずれかが1件以上存在する | `Client.IsActive` を `false` に更新する（レコードは削除しない） |

無効化された取引先は、[納品登録画面](delivery.md)・[売上報告入力画面](sales-report.md)の取引先選択肢から除外される。
