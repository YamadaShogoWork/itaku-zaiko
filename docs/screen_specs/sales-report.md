# 売上報告入力画面 仕様

対応モックアップ: [docs/mockups/sales-report.html](../mockups/sales-report.html)

## 1. 画面概要

FAXで受け取った期末在庫数を、取引先・対象年月単位で入力する画面。
入力した期末在庫数から売上点数・売上額を算出して表示する。

保存時に委託販売納品書（Excel、全項目）を出力し、月次繰越処理（期末在庫を翌月の `Delivery`・`IsCarryOver=true` として登録）を自動実行する（7章参照）。
翌月分の `SalesReport` が既に登録されている場合、当月分は編集できない（8章参照）。

---

## 2. 検索条件エリア

| 項目 | 種別 | データソース / 内容 |
|------|------|------|
| 取引先 | セレクトボックス | `Client` 全件を `ClientName` で表示（値は `ClientId`） |
| 対象年月 | 月入力 (`type=month`) | デフォルト値は「選択中の取引先の最終報告月（`SalesReport.YearMonth` の最大値）の翌月」 |
| 表示する（ボタン） | ボタン | 選択した取引先・対象年月で入力テーブルを再読込（GET） |

---

## 3. ステータスバー

| 項目 | データソース / 算出方法 |
|------|------|
| 取引先 | 選択中の `Client.ClientName`（検索条件のエコー表示） |
| 対象期間 | 選択中の対象年月を「YYYY年M月」形式で表示 |
| 売上合計 | 入力テーブルの「売上額」列の合計（合計行と同値） |
| 売上点数 | 入力テーブルの「売上点数」列の合計（合計行と同値） |

---

## 4. 入力テーブル

### 4.1 表示対象行

- 選択中の取引先（`Client`）の `ClientProduct` に登録されている `Product` のみを表示する。
- `Product` を `Color`（`Product.ColorId` で JOIN、null可）と結合する。
- `Product.ProductName` でグルーピングし、同一品名内は `ColorId` 昇順でソート（rowspanでまとめる）。

### 4.2 列定義

| 列 | データソース | 算出方法 |
|----|------|----------|
| 品名 | `Product.ProductName` | 色・サイズは連結しない（この画面のみの例外仕様） |
| 色 | `Color.ColorName` | `Product.ColorId` が null の場合は「—」表示 |
| 上代 | `Product.RetailPrice` | |
| 下代 | `Product.RetailPrice` × `ClientProduct.CommissionRate` | 小数になる場合もExcelの数式どおり丸めは行わない |
| 期首在庫 | `Delivery`（`ClientId`・`ProductId`一致、`IsCarryOver=true`）の `Quantity` | `DeliveredAt` が対象年月内のレコードを対象とする |
| 期間内納品数（最大4列） | `Delivery`（`ClientId`・`ProductId`一致、`IsCarryOver=false`）を納品日ごとに展開 | 列見出しは納品日（M/D形式）。対象年月内（`DeliveredAt` が対象月の範囲内）のレコードを対象とする |
| 納品計 | 計算値 | 期間内納品数の合計 |
| 期末在庫 | `SalesReport.ClosingStock`（`ClientId`・`ProductId`・`YearMonth`一致） | 入力欄。未入力時は空欄／プレースホルダ「—」 |
| 売上点数 | 計算値 | `期首在庫 + 納品計 − 期末在庫`。期末在庫未入力（かつ入力任意）の行は「—」表示 |
| 売上額 | 計算値 | `売上点数 × 下代`。売上点数が「—」の行は「—」表示 |

### 4.3 合計行

| 列 | 内容 |
|----|------|
| 品名〜期首在庫 | 結合して「合計」と表示 |
| 期間内納品数 | 空欄 |
| 納品計 | 各行の「納品計」の合計 |
| 期末在庫 | 「—」固定（入力列のため合計しない） |
| 売上点数 | 各行の「売上点数」の合計（未入力行は0として扱う想定） |
| 売上額 | 各行の「売上額」の合計 |

### 4.4 リアルタイム計算

期末在庫欄からフォーカスが外れた（blur）タイミングで、JavaScriptにより以下の表示を再計算・更新する。

- 該当行の「売上点数」「売上額」
- 合計行の「売上点数」「売上額」
- ステータスバーの「売上合計」「売上点数」

## 5. ボタン動作

| ボタン | 動作 |
|--------|------|
| 表示する | 選択中の取引先・対象年月の入力テーブルを読み込み直す。翌月分の `SalesReport` が既に存在する場合は8章の読み取り専用表示になる |
| クリア | 確認ダイアログを表示し、OKなら入力中の期末在庫欄をすべて空にする |
| 保存する | 確認ダイアログを表示し、OKなら入力チェック（6章参照）→ `SalesReport` にUPSERT → 月次繰越処理（7章）→ 委託販売納品書（Excel、全項目）をダウンロード → トーストで完了を通知し、同画面に留まる |

---

## 6. 入力チェック

- 期首在庫が1以上の商品 → 期末在庫の入力必須（未入力でエラー）
- 期首在庫が0の商品 → 期末在庫の入力は任意
- 翌月分（同一取引先・対象年月の翌月）の `SalesReport` が既に登録されている場合 → 保存不可（エラー）。8章の通り、この場合は画面表示時点で読み取り専用となり保存するボタン自体が表示されないため、通常はこのチェックに到達しない

---

## 7. 保存時の処理（月次繰越処理・Excel出力）

保存（UPSERT）が完了した後、以下を順に自動実行する。

1. **月次繰越処理**: 対象年月の翌月について、既存の繰越用 `Delivery`（`ClientId`一致・`IsCarryOver=true`・繰越元が当月であるもの）を削除し、入力テーブルの各行のうち期末在庫が1以上の商品について、期末在庫数を `Quantity` として翌月の `Delivery`（`IsCarryOver=true`）を新規登録する。期末在庫が0の商品は繰越レコードを作成しない（[delivery.mdの「数量0は登録しない」方針](delivery.md)と統一）。
2. **Excel出力**: 委託販売納品書（[3-3. Excel帳票出力](../requirements.md) の全項目版）を生成し、ブラウザでダウンロードする。期間内納品数・期首在庫・期末在庫・売上点数・売上額はすべて入力済みの値で出力する。

---

## 8. 翌月分が登録済みの場合（読み取り専用表示）

選択中の取引先・対象年月の翌月について `SalesReport` が1件以上存在する場合、入力テーブルを読み取り専用で表示する。

- ステータスバーの下に案内バナーを表示する: 「翌月（YYYY年M月）の売上報告が登録済みのため、この月の売上報告は編集できません。修正する場合は[売上報告履歴](sales-report-history.md)から翌月以降の売上報告を先に削除してください。」
- 期末在庫の入力欄（`.col-input`）はすべて disabled にし、保存済みの値を表示する
- 保存エリアの「クリア」「保存する」ボタンは非表示にする

---

## 9. 確定事項

- Excel出力ファイル名規則: `委託販売納品書_{取引先名}_{YYYYMM}.xlsx`（納品時・保存時・再出力の3タイミング共通。納品時の `YYYYMM` は納品日の年月）
