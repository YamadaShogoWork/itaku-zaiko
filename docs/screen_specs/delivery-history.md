# 納品履歴画面 仕様

対応モックアップ: 未作成（[delivery-history.html](../mockups/delivery-history.html) を新規作成予定）

## 1. 画面概要

これまでに登録された納品（`Delivery`）を一覧で確認し、編集・削除を行う画面。
編集は [納品登録画面](delivery.md) に取引先・納品日を指定して遷移し、既存数量のプレフィル（[delivery.md 4.2](delivery.md)）を利用する。

---

## 2. 検索条件

| 項目 | 種別 | 内容 |
|------|------|------|
| 取引先 | セレクトボックス | 「すべて」+ `Client` 全件。デフォルトは「すべて」 |
| 対象年月 | 月入力（`type=month`） | デフォルトは現在の年月 |
| 絞り込む（ボタン） | ボタン | 選択した条件で一覧を再読込（GET） |

---

## 3. 一覧

### 3.1 表示単位・列定義

1行 = 1登録単位（`ClientId` × `DeliveredAt` × `IsCarryOver` の組み合わせ）。

| 列 | データソース / 算出方法 |
|----|------|
| 納品日 | `DeliveredAt` |
| 取引先 | `Client.ClientName` |
| 品目数 | 同一グループに属する `Delivery` レコード数 |
| 合計数量 | 同一グループの `Delivery.Quantity` 合計 |
| 種別 | `IsCarryOver=false` → 「通常」バッジ、`true` → 「繰越」バッジ |
| 操作 | 編集・削除リンク（3.2参照） |

並び順: `DeliveredAt` 降順、同日内は `CreatedAt` 降順。

### 3.2 ボタン動作

| 操作 | 動作 |
|------|------|
| 編集 | [納品登録画面](delivery.md) に該当行の取引先・納品日をクエリパラメータ等で指定して遷移し、4.2のプレフィル付きで表示する |
| 削除 | 確認ダイアログを表示し、OKなら該当グループ（`ClientId` × `DeliveredAt` × `IsCarryOver`一致）の `Delivery` レコードをすべて削除し、一覧を再読込する |

繰越（`IsCarryOver=true`）行も編集・削除の対象とする。将来的に繰越行の編集・削除を禁止する場合は、操作列の表示制御のみで対応可能（実装への影響は小さい）。

---

## 4. 確定事項

- 検索条件「取引先」のデフォルト選択値: 未選択（空欄）。取引先を選ぶまでデータを表示しない。
- サイドバーのナビゲーション配置: 「業務」セクション内、「納品登録」の直後に「納品履歴」を配置する。
