# 委託販売 在庫管理システム ご説明資料

このシステムは、これまでExcelとFAXで行っていた「委託販売の在庫・売上管理」を、パソコンの画面で行えるようにするものです。

今日見ていただくのは、実際に動くシステムではなく**「画面のデザイン案（モックアップ）」**です。ボタンを押しても実際には何も起こりませんが、見た目や項目、操作の流れはほぼこのイメージで作る予定です。

「実際の仕事の流れと合っているか」「表示されている項目や言葉がわかりやすいか」を中心に見ていただけると助かります。

---

## 1. 今までとシステムでの違い

| | 今まで（Excel + FAX） | システム導入後 |
|---|---|---|
| 在庫・売上の記録 | 取引先ごとのExcelファイルに手入力 | パソコンの画面に入力すると自動で記録・計算 |
| 納品書の作成 | Excelで作成・印刷 | 画面に数量を入力するとExcelの納品書が自動で作られる |
| 売上点数・売上額の計算 | Excelの数式で手計算 | FAXで届いた期末在庫数を入力するだけで自動計算 |
| 翌月の期首在庫 | 前月のExcelを見ながら手入力 | 自動的に翌月の在庫としてシステムに引き継がれる |
| 全体の状況確認 | 各Excelファイルを開いて確認 | 「ダッシュボード」画面でまとめて確認できる |

基本的な業務の流れ（商品を納品 → 取引先がFAXで在庫数を返送 → 売上を計算）自体は今まで通りです。**入力と計算の手間を減らす**ことが目的です。

---

## 2. 全体の流れ

実際の業務の流れに沿って、システムをどう使うかをまとめます。

```
①商品を取引先に納品する
   └→ システムの「納品登録」画面で、納品した商品と数量を入力
   └→ そのまま委託販売納品書（Excel）が自動で作られる → 印刷して取引先へ

②取引先が在庫を数えて、納品書にその数を書いてFAXで返送
   （ここは今までと同じ）

③FAXで届いた期末在庫数を「売上報告入力」画面に入力
   └→ 売上点数・売上額を自動で計算して表示
   └→ 「保存する」を押すと…
        ・全項目入りの委託販売納品書（Excel）が自動で作られる
        ・入力した期末在庫数が、自動で翌月の「期首在庫」として
          システムに引き継がれる（来月はこの作業が省ける）

④いつでも「ダッシュボード」を見れば、
   今月の売上、在庫が少なくなっている商品などが一目でわかる
```

過去に登録した納品や売上報告は、「納品履歴」「売上報告履歴」の画面から後で確認・修正・帳票の再発行ができます。

## 3. 各画面の説明

モックアップは `docs/mockups` フォルダの中のHTMLファイルです。各見出しに対応するファイル名を記載しています。

### 3-1. ログイン画面（`login.html`）

パソコンを使う人を識別するための、最初のログイン画面です。基本的にこの方だけが使うシステムですが、念のためログインを必須にしています。

### 3-2. ダッシュボード（`dashboard.html`）

ログイン後に最初に表示される、全体の状況を確認する画面です。

- 今月の取引先数・納品件数・売上合計が一目でわかる summary（カード）
- 取引先ごとの売上ランキング（売上が多い順）
- **在庫アラート**：在庫が少なくなっている商品を「危険（残り1〜2点）」「警告（残り3〜5点）」で表示
- 最近登録した納品の履歴
- 「納品を登録する」「売上報告を入力する」のボタンから、すぐにそれぞれの画面に移動できる

### 3-3. 納品登録（`delivery.html`）

商品を取引先に納品したときに使う画面です。

1. 取引先と納品日を選ぶ
2. その取引先が取り扱っている商品（例：山菜図鑑Tシャツ 白／黒、山菜図鑑パーカー など）が一覧で表示される
3. 各商品の納品数量を入力する
4. 「登録してExcelを出力する」を押すと、入力内容が保存され、委託販売納品書（Excel）がダウンロードされる
   - この時点では期末在庫数・売上点数・売上額は空欄のまま出力されます（取引先が手書きでFAX返送するためです）

各商品の「現在庫」（だいたい今どのくらい在庫があるか）も一覧に表示されるので、納品数を決める際の参考になります。

### 3-4. 納品履歴（`delivery-history.html`）

これまでに登録した納品の一覧画面です。

- 取引先・対象年月で絞り込める
- 「いつ・どの取引先に・何品目・合計何個」納品したかが一覧で見られる
- 「編集」を押すと納品登録画面に内容を呼び出して修正できる
- 「削除」を押すとその納品の登録を取り消せる

### 3-5. 売上報告入力（`sales-report.html`）

取引先からFAXで届いた「期末在庫数」を入力する画面です。**月末ごとに使う、いちばん重要な画面**です。

1. 取引先と対象年月（例：2026年5月）を選ぶ
2. その取引先の取扱商品が一覧で表示される（品名・色・上代・下代・期首在庫・期間中に納品した数など）
3. FAXに書かれている期末在庫数を、各商品の「期末在庫」欄に入力する
4. 入力すると、その場で「売上点数」「売上額」が自動計算されて表示される
5. 「保存する」を押すと
   - 全項目入りの委託販売納品書（Excel）がダウンロードされる
   - 入力した期末在庫数が、自動的に翌月の期首在庫としてシステムに引き継がれる

※ なお、翌月分の売上報告を先に登録してしまった場合、その前月分は内容を変更できなくなります（翌月の期首在庫の元になっているデータのため）。修正したい場合は、翌月以降の売上報告を先に削除する必要があります。

### 3-6. 売上報告履歴（`sales-report-history.html`）

これまでに登録した売上報告の一覧画面です。

- 取引先・対象年月で絞り込める
- 「いつの・どの取引先の・何品目・売上点数・売上額」が一覧で見られる
- 「編集」を押すと売上報告入力画面に内容を呼び出して修正できる
- 「帳票」を押すと、その月の委託販売納品書（Excel、全項目入り）のプレビューが表示され、必要であれば再度Excelをダウンロードできる（紛失時の再発行用）
- 「削除」を押すとその月の売上報告を取り消せる（翌月分が登録済みの場合は削除できません）

### 3-7. その他の画面（マスタ管理など）

普段の作業（納品・売上報告）では使いませんが、商品や取引先の情報を最初に登録・整理しておくための画面です。

| 画面 | できること |
|---|---|
| 取引先（`client.html`） | 取引先（お店）の名前・FAX番号の登録、取り扱ってもらう商品と掛け率の設定 |
| 商品（`product.html`） | 商品名・色・上代（定価）・掛け率の登録 |
| 色（`color.html`） | 商品の色（白・黒・ネイビーなど）の登録・編集 |
| ユーザー管理（`users.html`） | システムにログインするためのユーザー名・パスワードの管理 |

---

## 4. モックアップの見方

1. `docs/mockups` フォルダの中の `.html` ファイルを、ダブルクリックでブラウザ（Safari/Chromeなど）で開いてください
2. 画面左側のメニューから他の画面にも移動できます
3. おすすめの見る順番：
   `dashboard.html` → `delivery.html` → `delivery-history.html` → `sales-report.html` → `sales-report-history.html`
   （慣れてきたら `client.html` や `product.html` なども見てみてください）
4. 表示されているデータ（取引先名・商品名・数字など）は**サンプル**です。実際のデータとは異なります
5. ボタンや入力欄は、見た目だけのデザインで、実際には保存などの動作はしません

---

## 5. 確認していただきたいこと

- 普段の「納品 → FAXで在庫数受け取り → 売上計算 → 翌月へ」という流れと、システムの流れ（2章）が合っているか
- 各画面に表示されている項目・用語（品名、上代、下代、期首在庫、期末在庫など）が、普段使っている言葉とズレていないか
- 入力する項目（納品数量、期末在庫数など）に、足りないものや余分なものがないか
- 画面の見やすさ・操作のしやすさについて気になる点
