# 自動テスト仕様書

`node test_flow.js` で実行する Playwright ベースの自動テスト。  
ヘッドレスChromiumでブラウザ操作を再現し、47件のテストケースを順番に実行する。

---

## 実行方法

```bash
# アプリが起動済みであること（http://localhost:5038）
node test_flow.js
```

- スクリーンショットは `C:/Work/Zaiko/screenshots/` に保存される
- テスト結果はコンソールに `✅ PASS` / `❌ FAIL` で出力される
- テストは上から順に依存関係があるため、途中で失敗すると後続が連鎖して失敗する場合がある

## 繰り返し実行時の注意

テストはDBをリセットせずに繰り返し実行できる設計になっている。各セクションの冒頭でクリーンアップ処理が入っており、前回の実行データを削除してから始める（testuser・SalesReport）。

## 自動テストでカバーしていない項目

以下は `docs/manual_test.md` で手動確認する。

- 「ログイン状態を保持する（60日間）」の動作
- 同名重複インライン警告（フォーカスアウト時）
- 納品履歴の「帳票」ボタン（直接ダウンロード）
- 検索クリアボタン・空検索時の全件表示
- 100件ページング（大量データ時）
- 取引先編集の商品D&D並び替え（SortOrder）
- 売上報告入力の対象年月デフォルト（15日境界）
- 売上報告入力の未入力行エラー
- Excelファイルの内容・書式の詳細確認
