# 委託販売在庫管理システム 要件定義

## 1. プロジェクト概要

親の自営業（ハンドメイド商品の製造・販売）における委託販売の手作業をシステム化する。
現在はExcelで管理している取引先ごとの在庫・売上管理を、Webアプリケーションとして構築し、帳票（委託販売納品書）をExcelで出力できるようにする。

---

## 2. ビジネス背景

- 商品（Tシャツ・パーカー・トートバッグ・手ぬぐい・サコッシュ等）を複数の取引先に委託販売している
- 取引先×商品の組み合わせごとに掛け率（上代に対する下代の割合）が異なる
- 納品が発生すると、納品書（Excel、委託販売納品書と同じ様式）に納品日・納品数量を記載し、期末在庫数・売上点数・売上額は空欄のまま取引先に送付する
- 取引先はその納品書を印刷し、期末在庫数を手書きした上で月末にFAXで返送する（FAXで届くのは期末在庫数のみ）
- 期末在庫は翌月の繰越として翌月最初の納品扱いになる
- 月の途中で追加納品が発生することがある

## 3. 機能要件

### 3-1. マスタ管理

| 機能 | 内容 |
|------|------|
| 取引先管理 | 取引先の登録・編集・削除（名前・FAX番号）。取引先ごとの取扱商品（`ClientProduct`、商品・掛け率）の登録・編集も行う。関連データがある取引先は削除の代わりに無効化（`IsActive=false`）する |
| 商品管理 | 商品の登録・編集・削除（商品名・色・上代・掛け率）。品名単位で上代・掛け率（標準値）を共通入力し、色は複数選択して色ごとに`Product`レコードを一括登録する。サイズはDB設計上保持するが、本画面のUIには項目を設けない。関連データ（`Delivery`/`SalesReport`/`ClientProduct`）がある商品は削除できない（無効化フラグは持たない） |
| 色マスタ管理 | 色の登録・編集・削除（一覧の行をインライン編集する形式）。関連する`Product`がある色は削除できない（無効化フラグは持たない） |

#### 商品名の表示仕様

- `Product.ProductName` は色・サイズを含まない基本名称とし、複合語の間にスペースは入れない（例：`山菜図鑑Tシャツ`、`山菜図鑑パーカー`）
- 一覧・履歴等の単一表示では、`ColorId`/`SizeId` が設定されている場合に `ProductName + 半角スペース + ColorName（+ 半角スペース + SizeName）` を表示時に連結する
  - 例：`山菜図鑑Tシャツ 白`、`山菜図鑑パーカー 黒 L`
- 色・サイズが未設定の商品（トートバッグ・手ぬぐい等）は `ProductName` のみ表示する（ProductName自体に含まれる語句・スペースはその商品名の一部）
- 例外：売上報告入力画面のように色ごとに行を分ける入力グリッドでは、品名（連結前のProductName）と色を別列に分けて表示する
- `Product.SizeId` は現状未使用（null想定）であり、表示・連結の対象外とする

### 3-2. トランザクション管理

| 機能 | 内容 |
|------|------|
| 納品登録 | 取引先・納品日を選択し、その取引先の取扱商品をグリッド形式で表示。商品ごとに納品数量を入力して一括登録する（取引先×納品日の単位で1件の納品として扱う）。登録と同時に委託販売納品書（Excel、期末在庫数等は空欄）を出力できる |
| 売上報告入力 | FAXで受け取った期末在庫数を月次で登録する。保存時に委託販売納品書（Excel、月末版）を出力する |
| 月次繰越処理 | 売上報告入力画面の保存時に自動実行。期末在庫を翌月の繰越納品（IsCarryOver=true）として登録する |

#### 納品登録画面の品名表示ロジック

- 売上報告入力画面の品名表示ロジック（後述）と同じロジックを適用する（対象取引先の `ClientProduct` に登録された商品に限定し、`ProductName` でグルーピング、グループ内は `ColorId` 昇順で表示）

#### 売上報告入力画面の品名表示ロジック

- 表示対象商品は、選択中の取引先の `ClientProduct` に登録された商品に限る
- 商品は `Product.ProductName` でグルーピングし、同一品名内の色違いはrowspanでまとめて表示する（カテゴリ見出しは設けず、フラットな一覧とする）
- グループ内の色（行）の表示順は `ColorId` 昇順とする

#### 売上報告入力画面の入力チェック

- 期首在庫（`Delivery` の `IsCarryOver=true` レコードの `Quantity`）が存在する商品は、期末在庫の入力を必須とする（未入力のまま保存しようとした場合はエラー）
- 期首在庫が存在しない商品は、期末在庫の入力を任意とする（FAXで報告がないことがあるため）
- 翌月分の `SalesReport`（同一取引先・対象年月の翌月）が既に登録されている場合、当月分の保存（編集）はできない（翌月の期首在庫の根拠データが変わってしまうため）。修正する場合は翌月以降の売上報告を先に削除する必要がある

### 3-3. Excel帳票出力

帳票出力は以下の3種類があり、いずれも「委託販売納品書」形式（同じExcelテンプレート）を使用する。

1. **納品時の納品書出力**（納品登録画面から、登録対象の取引先・納品日についてその場で出力）
   - 期間内納品数列のうち、該当納品日の列に今回の納品数量を記載する
   - 期首在庫数（繰越分）は参考として記載する
   - 期末在庫数・売上点数・売上額は空欄（取引先が手書きでFAX返送するため）
   - 月次繰越処理は実行しない
2. **売上報告保存時の確定納品書出力**（売上報告入力画面）
   - 売上報告入力画面で取引先・対象年月を選択し、期末在庫数を入力して保存すると、その場で全項目入りの帳票を出力する
   - 操作フロー：取引先・対象年月入力 → 表示する → 期末在庫数を入力 → 保存する（確認ダイアログ）→ 保存 + Excelファイル出力 + 月次繰越処理を自動実行
   - 期間内納品数の列は最大4列（納品回数の上限）
3. **売上報告履歴画面からの再出力**（売上報告履歴画面）
   - 一覧の各行の「帳票」リンクからモーダルを開き、保存済みの `SalesReport` データから2.と同内容の帳票プレビューを表示する
   - 操作フロー：売上報告履歴一覧 → 「帳票」リンク → モーダルでプレビュー確認 → Excel出力ボタン → Excelファイル出力
   - 月次繰越処理は実行しない
   - 月次繰越処理は実行しない（既に保存時に実行済みのため）

#### 帳票の列構成

| 列 | 内容 | 算出方法 |
|----|------|----------|
| 品名 | 商品名 | Product.ProductName |
| 色 | 色名 | Color.ColorName |
| 上代（税込） | 定価 | Product.RetailPrice |
| 下代（税込） | 卸値 | RetailPrice × ClientProduct.CommissionRate |
| 期首在庫数 | 繰越納品の数量 | Delivery（IsCarryOver=true）の Quantity |
| 期間内納品数 | 各納品日ごとの列 | Delivery（IsCarryOver=false）を納品日ごとに展開 |
| 期間内納品計 | 期間内納品の合計 | Delivery（IsCarryOver=false）の Quantity 合計 |
| 期末在庫数 | FAXで受け取った実数 | SalesReport.ClosingStock |
| 売上点数 | 計算値 | 期首在庫 + 期間内納品計 − 期末在庫 |
| 売上額 | 売上金額 | 売上点数 × 下代 |

### 3-4. ダッシュボード

| 機能 | 内容 |
|------|------|
| サマリー表示 | 取引先数・今月の納品件数・今月の売上合計・在庫アラート件数を表示 |
| 取引先別売上ランキング | 当月の取引先別売上額を一覧表示 |
| 在庫アラート | 現在庫が少ない商品（取引先×商品単位）を警告表示 |
| 最近の納品履歴 | 直近のDelivery登録を一覧表示 |

#### 在庫アラートのロジック（暫定）

- **現在庫の算出方法**：
  - 対象（取引先×商品）について、直近のSalesReportが存在する場合：
    `現在庫 = その SalesReport.ClosingStock + （対象月より後に登録された IsCarryOver=false の Delivery の Quantity 合計）`
  - 直近のSalesReportが存在しない場合：
    `現在庫 = それまでに登録された Delivery（IsCarryOver問わず）の Quantity 合計`
- **アラート区分**：
  - 現在庫が 1〜2 → 危険（赤）
  - 現在庫が 3〜5 → 警告（橙）
  - 現在庫が 6 以上 → アラートなし
- 上記は暫定ロジックであり、運用しながら閾値や算出方法を見直す可能性がある。

## 4. 非機能要件

| 項目 | 内容 |
|------|------|
| フレームワーク | ASP.NET Core MVC (.NET 10) |
| DB | SQLite + Entity Framework Core |
| 認証 | ASP.NET Core Identity（ログイン必須、全ユーザーが全機能を利用可能、ロールによる権限分けは行わない） |
| ユーザー管理 | 管理画面でユーザーの登録・削除が可能（基本的に1人での利用を想定するが、ユーザー登録機能自体は提供する） |
| 開発環境 | VS Code / Windows |
| 実行環境 | 利用者PC（macOS）。1台のPC内で完結する運用（外部公開・サーバーへのホスティングは行わない） |
| Excel出力 | ClosedXML 等のライブラリを使用 |

## 5. データベース設計

### 共通カラム（全テーブル）

全テーブルに以下の監査カラムを持たせる。

| カラム | 型 | 説明 |
|--------|----|------|
| CreatedAt | DateTime | 作成日時 |
| CreatedBy | string | 作成者（ユーザー名） |
| UpdatedAt | DateTime | 更新日時 |
| UpdatedBy | string | 更新者（ユーザー名） |

### テーブル一覧

#### Client（取引先マスタ）

| カラム | 型 | 説明 |
|--------|----|------|
| ClientId | int (PK) | |
| ClientName | string | 取引先名 |
| FaxNumber | string | FAX番号 |
| IsActive | bool | 有効フラグ（デフォルトtrue）。無効な取引先は納品登録・売上報告入力画面の選択肢から除外される |

取引先削除時、`Delivery`/`SalesReport`/`ClientProduct`等の関連データが存在しない場合は物理削除する。存在する場合は削除せず`IsActive=false`に切り替える（論理削除）。

#### Color（色マスタ）

| カラム | 型 | 説明 |
|--------|----|------|
| ColorId | int (PK) | |
| ColorName | string | 色名（例：白、黒、アッシュ） |

#### Size（サイズマスタ）

| カラム | 型 | 説明 |
|--------|----|------|
| SizeId | int (PK) | |
| SizeName | string | サイズ名（例：S、M、L、XL） |

#### Product（商品マスタ）

| カラム | 型 | 説明 |
|--------|----|------|
| ProductId | int (PK) | |
| ProductName | string | 商品名 |
| ColorId | int? (FK) | 色（NULL可） |
| SizeId | int? (FK) | サイズ（NULL可） |
| RetailPrice | int | 上代（税込） |
| CommissionRate | decimal | 掛け率の標準値（例：Tシャツ系0.75、パーカー系0.8）。`ClientProduct.CommissionRate` の初期値として使用 |

`SizeId`は現状未使用（null想定）。`Size`マスタ・`Product.SizeId`カラムはDB設計上保持するが、管理用UI（商品管理画面・サイズマスタ管理画面）は現状設けない（常にnullで登録される）。将来サイズ管理が必要になった時点で別途UIを追加する。

#### Delivery（納品記録）

| カラム | 型 | 説明 |
|--------|----|------|
| DeliveryId | int (PK) | |
| ClientId | int (FK) | 取引先 |
| ProductId | int (FK) | 商品 |
| Quantity | int | 納品数量 |
| DeliveredAt | DateTime | 納品日 |
| IsCarryOver | bool | 繰越フラグ（true=期末在庫からの繰越） |

納品登録画面で1回に登録する「納品（取引先×納品日）」は、`ClientId` と `DeliveredAt`（と `IsCarryOver=false`）の組み合わせで識別する。新たな「納品バッチ」テーブルは設けない。

#### SalesReport（売上報告）

| カラム | 型 | 説明 |
|--------|----|------|
| SalesReportId | int (PK) | |
| ClientId | int (FK) | 取引先 |
| ProductId | int (FK) | 商品 |
| YearMonth | string | 対象年月（例：2026-05） |
| ClosingStock | int | 期末在庫数（FAX入力値） |

#### ClientProduct（取扱商品）

| カラム | 型 | 説明 |
|--------|----|------|
| ClientProductId | int (PK) | |
| ClientId | int (FK) | 取引先 |
| ProductId | int (FK) | 商品 |
| CommissionRate | decimal | 掛け率（取引先×商品ごと）。下代の算出に使用する実際の値。登録時は `Product.CommissionRate` を初期値としてプレフィルし、その取引先だけ異なる場合は上書きする |

取引先ごとに「取り扱う商品」を登録するための中間テーブル。売上報告入力画面では、対象取引先の `ClientProduct` に登録された商品のみを表示する。

### インデックス・ユニーク制約

| テーブル | 対象カラム | 種別 | 理由 |
|----------|-----------|------|------|
| Delivery | `(ClientId, ProductId, IsCarryOver, DeliveredAt)` | 複合インデックス | 現在庫算出・帳票生成で頻繁に検索 |
| SalesReport | `(ClientId, ProductId, YearMonth)` | ユニーク制約 + インデックス | 同一取引先×商品×年月のUPSERT・重複防止 |
| ClientProduct | `(ClientId, ProductId)` | ユニーク制約 + インデックス | 同一取引先×商品の二重登録防止 |

## 6. 画面一覧（予定）

| 画面 | 説明 |
|------|------|
| ダッシュボード | サマリー情報・在庫アラート・最近の納品履歴等を表示するトップ画面 |

| 商品一覧・登録・編集 | Productマスタ管理 |

## 7. 未決事項

- [x] 商品にサイズ・色がない場合の帳票表示方法 → 品名のみ表示
- [x] 月次繰越処理のタイミング → 売上報告入力画面の保存時に自動実行（売上報告履歴画面の帳票プレビュー（モーダル）は再出力専用とし、繰越処理は実行しない）
- [x] 帳票の「期間内納品数」列数 → 最大4列
- [x] 売上報告入力画面で未入力（期末在庫未入力）の商品がある場合の保存可否 → 期首在庫が存在する商品は入力必須（未入力ならエラー）、期首在庫が存在しない商品は任意
- [x] 納品履歴一覧の表示単位（商品ごとの行のまま vs 納品＝取引先×納品日でまとめる） → 当面作らない（実装対象外）
- [x] 【重要】掛け率（下代 = 上代 × 掛け率）を `Client`・`Product` どちらに持たせるか
  - 親に確認した結果、掛け率は**取引先×商品の組み合わせごと**に存在する（実データの2取引先では商品カテゴリ依存の値が偶然一致していた）
  - → `Client.CommissionRate` は廃止。`Product.CommissionRate`（商品ごとの標準値、例：Tシャツ系0.75・パーカー系0.8）を新設し、`ClientProduct.CommissionRate`（実際の算出に使う取引先×商品ごとの値、登録時は `Product.CommissionRate` をプレフィル）を新設
  - 下代 = `Product.RetailPrice × ClientProduct.CommissionRate`
- [x] 下代が小数になる場合の丸め方（四捨五入／切り捨て／切り上げ）
  - Excelの計算と同じ仕様とする（`=上代*掛け率` の数式どおり、ROUND等は行わず計算結果をそのまま使用）
- [ ] 実行環境がmacOSであることに伴う配布・起動方法（開発はWindows、利用者PCはmacOS）
  - .NETランタイムのインストールが必要か、self-contained publish（macOS向けに単体実行ファイルを作成）にするか
  - 起動方法（ターミナル起動／ダブルクリックで起動するスクリプトやアプリ化等）
  - SQLiteのDBファイル・Excel出力先のパスをOS依存しない形（相対パス・`Path.Combine`等）で扱う点に注意
